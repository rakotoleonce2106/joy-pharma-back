## âš™ï¸ Key Business Rules Explained

### 1. Inventory-Based Operations
Stores can ONLY accept or suggest products that exist in their `StoreProduct` inventory. This ensures:
- Accurate stock management
- No manual price entry errors
- Automatic price fetching

### 2. Automatic Pricing
When a store accepts an order item:
- Price is automatically fetched from `StoreProduct` table
- NO manual price input required
- Prevents pricing errors

### 3. Order Item Workflow
```
Customer Orders â†’ Store Receives â†’ Store Actions:
  â†’ Accept (with notes)
  â†’ Refuse (with reason)
  â†’ Suggest Alternative (requires admin approval)
```

### 4. Dual Pricing System
- `totalAmount`: Based on catalog prices
- `storeTotalAmount`: Based on store-specific prices (accepted items)
